# Roadmap

This public roadmap is intentionally high-level. It describes the project
boundary without publishing the private order of work, internal operating plan,
or product strategy.

For shipped work, read `CHANGELOG.md`.

For public work that is intentionally open to contributors, read GitHub issues.

For private planning, use the private maintenance journal described in
`docs/MAINTENANCE_MANUAL.md`.

## Public Direction

- Keep the default deployment local-first and safe.
- Keep fake demo data separate from private user data.
- Keep Personal OS and Personal Wiki as one product with separate runtime
  services.
- Keep agent-facing APIs explicit, documented, and test-covered.
- Keep review and evidence in the product boundary.
- Keep private notes, private task history, server inventories, and real
  deployment details out of the public repository.

## Not Goals For The Current Release

- Hosted SaaS.
- Multi-tenant organization management.
- OAuth/user-account system.
- Fully autonomous agents that can bypass review.
- Publishing real private vaults or server inventories.
- Replacing Obsidian, Logseq, Notion, or your editor.

The public repository should show what is safe to use and verify. It should not
publish the private sequence of product bets.
