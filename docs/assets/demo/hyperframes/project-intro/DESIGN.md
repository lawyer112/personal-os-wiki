# Personal OS + Personal Wiki Project Intro Design

## Style Prompt

A precise product introduction film for a local-first agent workbench. The visual language is operating-system calm: structured grids, audit trails, task rails, Markdown panels, and API handoff lines. It should feel like a serious builder tool, not a marketing landing page or installer walkthrough.

## Colors

- `#f7f9fb` canvas: clean public-demo background.
- `#111827` ink: primary text and structural lines.
- `#2563eb` OS blue: execution state, task ownership.
- `#0f766e` wiki teal: durable knowledge and evidence.
- `#7c3aed` agent violet: worker protocol and handoff.
- `#f59e0b` review amber: human review and decision points.
- `#e5e7eb` rule gray: separators, grids, quiet UI borders.

## Typography

- Primary: `"Microsoft YaHei", "PingFang SC", "Noto Sans CJK SC", Arial, sans-serif`.
- Use strong but compact Chinese product copy. Keep titles large only on the opening and closing frames; use dense dashboard-scale labels inside panels.
- Letter spacing stays `0`.

## Motion

- Mechanical handoffs: short slides, line reveals, evidence cards locking into place.
- No infinite loops. Any repeated movement must be finite and timeline-bound.
- Avoid bokeh, gradient orbs, stock imagery, or installer/package visuals.

## What NOT to Do

- Do not describe Windows/macOS/Linux installers or package formats.
- Do not expose real tokens, real hostnames, private Wiki content, or private task history.
- Do not present the project as a generic note app.
- Do not make the human disappear from the review loop.
