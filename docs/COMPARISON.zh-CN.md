# Personal OS vs Agent 长期记忆 vs Obsidian 任务插件

一句话：

```text
Agent 长期记忆记住偏好。
Obsidian 插件组织笔记。
Personal OS + Personal Wiki 把知识变成可复核的 Agent 工作。
```

## 真正的问题

真正的 Agent 工作不能把聊天记录当真相来源。

Agent 可能记得“我们聊过这个方向”，但它通常不知道：

- 当前打开的任务是什么；
- 谁认领了任务；
- 认领什么时候过期；
- 提交了什么证据；
- Reviewer 是否通过；
- 今天应该提醒什么；
- 哪篇 Wiki 笔记支撑这个判断。

所以这个项目不只是想做一个更好的笔记工具。它是给 Agent 用的任务控制面，
Markdown Wiki 是它的长期上下文层。

## 对比

| 能力 | Agent 长期记忆 | Obsidian 任务插件 | Personal OS + Personal Wiki |
| --- | --- | --- | --- |
| 记住用户偏好 | 强 | 弱 | 交给 Agent 自带记忆 |
| 保存长期笔记 | 弱到中等 | 强 | 强，基于 Markdown |
| 双链和知识图谱 | 通常没有 | 强 | 有，来自 Wiki 笔记和 graph data |
| 明确任务归属 | 通常没有 | 有限 | 有，支持任务认领 |
| Agent 心跳和租约 | 没有 | 没有 | 有 |
| 提交证据后复核 | 没有 | 没有 | 有 |
| Agent 读写 token 边界 | 通常没有 | 不适用 | 有 |
| 生成提醒 payload | 通常没有 | 有限 | 有 |
| 多 Agent 共用 API | 通常没有 | 没有 | 有 |
| 私有运行数据不进 Git | 取决于实现 | 取决于使用者 | 仓库边界里明确设计 |

## 各自适合什么

Agent 长期记忆适合放稳定事实：

- 用户偏好；
- 命名习惯；
- 固定操作规则；
- 长期约束。

Obsidian 或笔记插件适合人类浏览知识：

- 看笔记；
- 手动双链；
- 写长文研究；
- 个人日志。

Personal OS + Personal Wiki 适合需要执行状态的场景：

- Agent 可以认领的任务；
- 可复核提交；
- 证据链接；
- 今日计划；
- 提醒投递；
- 跨 Agent、跨会话保留的项目状态。

## 为什么这不是另一个第二大脑

第二大脑回答：

```text
我知道过什么？
```

这个项目回答：

```text
下一步该推进什么，谁负责，什么证据证明它真的动了？
```

这就是项目的窄切口。它应该继续优化 Agent 执行闭环，而不是卷进泛泛的个人知识管理。
